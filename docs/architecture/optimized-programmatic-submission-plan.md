# Programmatic Submission Plan (OSS Sanitized)

This public plan describes a synthetic, safe-by-default filing package engine. It intentionally omits proprietary reverse-engineering notes, private fixtures, production endpoints, credentials, and official application artifacts.

## Scope

- Render form payloads from data-driven templates.
- Package rendered payloads through a deterministic transform for testability.
- Queue dry-run/live-gated submission jobs.
- Store local profiles and settings.
- Match receipt-like messages to durable submission records.

## Current built-in form coverage

Only a synthetic 1601C fixture is included. Additional forms require clean-room templates and mappings.

## Safety

Live submission is disabled unless explicit live flags and operator-provided configuration are present. No production credentials or private taxpayer data are included.
